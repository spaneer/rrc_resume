RRC States & Transitions

NSA and SA – Scenario-Based Web Timeline

1. Power ON → IDLE
NSASA
Transition: UE Off → RRC_IDLE
UE powers on, searches and synchronizes with the network, reads system info, and registers with the core.
2. User Data Trigger
NSASA
Transition: RRC_IDLE → RRC_CONNECTED
User opens an app or sends data. UE performs RRC Setup procedure and enters the connected state.
3. Network Paging
NSASA
Transition: RRC_IDLE → RRC_CONNECTED
Network pages the UE. UE resumes from idle and reconnects to handle downlink data or signaling.
4. Connected → Inactive
SA only
Transition: RRC_CONNECTED → RRC_INACTIVE
In 5G SA, UE may be moved to RRC_INACTIVE for faster future resume and to conserve battery.
5. Resume from Inactive
SA only
Transition: RRC_INACTIVE → RRC_CONNECTED
When data is required again, UE uses the RRC Resume procedure, avoiding full setup and reducing latency.
6. Inactivity Timer Expired
NSASA
Transition: RRC_CONNECTED → RRC_IDLE or RRC_INACTIVE
If the user is inactive for a while, the UE releases resources and moves back to idle or inactive (in SA).
7. Mobility / Handover
NSASA
Transition: RRC_CONNECTED → RRC_CONNECTED (new cell)
While moving, UE hands over to a different cell or frequency, remaining in connected state throughout.
8. ENDC Activation (Dual Connectivity)
NSA onlyLTE Anchor
Transition: LTE RRC_CONNECTED + Add NR
In NSA, UE remains in LTE RRC_CONNECTED. NR is added for high-speed data; control remains over LTE.
